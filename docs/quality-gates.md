# Quality Gates

## Gate A: Problem Value

Pass:

- clear research question
- measurable metrics
- at least two baselines
- available data or benchmark
- minimum experiment loop exists

Reject:

- only "use LLM to improve X"
- no baseline
- no clear metric
- mostly subjective evaluation
- requires compute budget that is not available

## Gate B: Idea Novelty

Pass:

- clear difference from existing methods
- difference can be isolated experimentally
- not only prompt engineering
- not only swapping models
- not only parameter tuning

Reject:

- prior work already does the core idea
- contribution is only engineering glue
- method needs unfair baselines to win

## Gate C: Implementation Trust

Pass:

- tests pass
- smoke test passes
- baseline and method share the same data pipeline
- metrics schema is identical across methods
- failed runs are recorded

Reject:

- code runs but has no tests
- baseline preprocessing differs
- result tables are manually filled
- logs are missing

## Gate D: Experiment Trust

Pass:

- main experiment complete
- ablations complete
- negative results preserved
- repeated runs or statistical checks exist
- hardware and environment recorded

Reject:

- only best seed shown
- only favorable datasets shown
- failed runs deleted
- variance not reported

## Gate E: Claim Trust

Pass:

- every claim exists in `70_claims/claims.yaml`
- every number has a source
- every figure can be regenerated by script
- limitations are explicit

Reject:

- claim exceeds experiment scope
- inconclusive result written as supported
- citation is missing or irrelevant
